• welcome to bubble-tree

    • purpose

      Use bubble-tree to dress up content-light sites such as personal home pages and placeholder sites.

      If you can code a few nested <ul> lists, you can make a site like this one.

    • the UX

      • touching or clicking a circle

        • Touching or clicking a circle zooms in our out on the list so that the circle fills the viewport or frame.

        • When zoomed in, a link for returning to the top appears in the top left. It disappears when touched.

        • Offsite links are disabled when they are very small. They become active when the user zooms in close enough.

      • portrait vs landscape

        Nested <ul> lists are displayed to the right of their parent <li> elements. They add width, but not height, to their parents.

        At runtime, JavaScript sizes the list to fit the screen, regardless of the orientation of either.

        • Use one <li> element in your top-level <ul> list for a landscape orientation dominated by one large circle.

        • Use two or more <li> elements for a portrait orientation.

      • Customizable

        By default, background images are shown centered and scaled to cover the circle.

    • add

      Add the file to your project and link to it from your html file

    • add
      bubble-tree.js

      Add the file to your project and link to it from your html file

    • make
      <ul id="bubbleTree">

      Create well-formed nested <ul> lists and give the highest UL tag the id "bubbleTree".

    • add the <a> tag

      add
      <a href="#" id="bubbleTopLink" class="hidden">

    • should be done

      Congratulations! Now you're ready to make it your own.

  • limitations

    • <li> tag can't contain plain text

      You need to put the content of the <li> tag inside a tag such as P, H1, H2, etc.

    • Maximum 5 items per list

      This limitation could be easily overcome by building out more CSS classes, but I couldn't be bothered. It should be obvious if you are so inclined.
